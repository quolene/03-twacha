03-Twacha
I was trying to buy a sunscreen. Ended up with seven tabs open, three conflicting reviews, and no idea what half the ingredients meant. Spreadsheets wasn't it. Googling each ingredient separately definitely was time and effort consuming.
So: search once, see all five prices, understand what's actually in it. That's the whole thing.
Search by product, brand, or ingredient. Filter by skin type. Enter your pincode and see how fast it reaches you. Ask the guide if you're confused about something — it won't upsell you.
No paid rankings. No affiliate links. No brand paying to appear first.
Live: https://quolene.github.io/03-twacha/
Could have built a full database, real-time scraping, user accounts, a recommendation engine. Didn't.
The actual problem was simpler — people don't know what they're buying or where to buy it cheaper.
Prices are representative — updated weekly by hand. Live API integration is on the roadmap.
Built during my internship at Silverpoints. CogSci background, so the information architecture and the copy are as deliberate as the code.
